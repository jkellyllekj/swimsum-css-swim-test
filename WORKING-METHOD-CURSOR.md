# Working Method - CSS Swim Test (Cursor)

Last updated: 2026-03-27

## Core rules

1. Keep this project separate from `swimgen2`.
2. Update `project-state.md` as the memory for decisions and progress.
3. Prefer small, testable changes.
4. Avoid broad refactors unless explicitly requested.
5. Keep instructions concise and concrete.

## Session flow

1. Read `project-state.md` first.
2. Confirm current task in one sentence.
3. Implement the smallest useful change.
4. Validate quickly.
5. Record outcome in `project-state.md` when requested.

## Scope boundaries

- This repo is for CSS Swim Test only.
- Do not change SwimSum main app / Stopwatch code from this workspace.
- Cross-project references belong in `REFERENCE-LINKS.md`.

## Documentation

- Use the append-only **Progress log** section in `project-state.md` for dated implementation, hosting, and product decisions. Do not delete or shorten earlier log entries when adding new ones.
